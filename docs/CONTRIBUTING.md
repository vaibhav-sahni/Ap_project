# Contributing

## Workflow

1. Fork and create a feature branch.
2. Keep changes focused and incremental.
3. Include tests for server logic where practical.
4. Open a PR to `main` with a clear description and screenshots for UI changes.

## Commit Messages

- Use imperative tense (e.g., "Add", "Fix", "Refactor").
- Reference related issues.

## Code Review

- Address comments promptly.
- Explain trade-offs and chosen design.

## Documentation

- Update `README.md` or `docs/` when adding/modifying features.
